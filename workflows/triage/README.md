# Issue Triage Workflow

A systematic workflow for triaging repository issue backlogs with AI-powered recommendations and bulk operations support. Supports both GitHub and Jira.

## Overview

This workflow separates **data generation** from **report rendering** for deterministic, bug-free output:

1. **AI agent** analyzes issues and produces `triaged_issues.json`
2. **`render-report.py`** reads the JSON and renders HTML + Markdown reports from templates
3. **`bulk-operations.sh`** executes approved operations against GitHub or Jira

Templates are never modified or regenerated by the AI agent.

## Features

### Interactive HTML Report

- **Sortable Columns**: Click any column header to sort
- **Simulation Mode**: Preview how recommendations affect the backlog
- **Filters**: Filter by recommendation, type, priority, or search
- **Bulk Operations**: Select issues and generate operations JSON
- **Platform-aware**: Adapts links and commands for GitHub or Jira

### Recommendations

- **CLOSE**: Invalid, obsolete, or duplicate issues
- **FIX_NOW**: Critical issues or quick wins
- **BACKLOG**: Valid issues for future work
- **NEEDS_INFO**: Blocked on additional information
- **DUPLICATE**: Duplicate of existing issue
- **CBA_AUTO**: Can be fixed automatically by a codebase agent
- **ASSIGN**: Ready to be assigned to a developer
- **WONT_FIX**: Valid but won't be addressed

### Custom Labels

The AI agent can add arbitrary labels per issue via the `customLabels` field (e.g., `ai-fixable`, `ai-triaged`). These are included as additional operations in the bulk script output.

### Bulk Operations

Execute bulk operations on GitHub or Jira:

```bash
# GitHub
./scripts/bulk-operations.sh --backend github --repo owner/repo --operations operations.json

# Jira
./scripts/bulk-operations.sh --backend jira --operations operations.json

# Preview first
./scripts/bulk-operations.sh --dry-run --backend github --repo owner/repo --operations operations.json
```

Supports: closing issues, adding/removing labels, assigning issues, linking duplicates, adding comments.

## Quick Start

### Option 1: Manual Triage (Local)

1. **Start workflow:**

   ```bash
   ambient session create --workflow triage
   ```

2. **Provide repository (GitHub):**

   ```text
   Triage the backlog for https://github.com/owner/repo
   ```

3. **Or provide JQL (Jira):**

   ```text
   Triage the issues from this JQL: project = MYPROJECT AND status in (Backlog, New)
   ```

4. **Open generated report:**

   ```bash
   open artifacts/triage/report.html
   ```

### Option 2: Automated Triage (GitHub Actions)

1. **Copy workflow to your repo:**

   ```bash
   cp .github/workflows/triage-report.yml /your/repo/.github/workflows/
   ```

2. **Commit and push:**

   ```bash
   git add .github/workflows/triage-report.yml
   git commit -m "Add automated triage workflow"
   git push
   ```

3. **Workflow runs automatically monthly or manually via Actions tab**

4. **Download artifact and open report.html**

## Files

- `.ambient/ambient.json` - Workflow definition
- `templates/report.html` - Interactive HTML template (placeholders only)
- `templates/triage-report.md` - Markdown template (placeholders only)
- `scripts/render-report.py` - Deterministic report renderer
- `scripts/bulk-operations.sh` - Bulk operations executor
- `.github/workflows/triage-report.yml` - GitHub Actions workflow
- `FIELD_REFERENCE.md` - Complete field and schema reference

## Outputs

Generated in `artifacts/triage/`:

- `triaged_issues.json` - Structured triage data (AI agent output)
- `report.html` - Interactive dashboard (rendered from template)
- `triage-report.md` - Markdown report (rendered from template)

## Data Flow

```text
Issues (GitHub/Jira)
       |
       v
  AI Agent analyzes
       |
       v
triaged_issues.json   <-- AI's only data output
       |
       v
render-report.py      <-- reads JSON + templates
       |
       v
report.html + triage-report.md
       |
       v
User reviews in browser
       |
       v
operations.json       <-- generated by HTML UI
       |
       v
bulk-operations.sh    <-- executes against GitHub/Jira
```

## Requirements

- Python 3.x (for render script)
- GitHub CLI (`gh`) for GitHub issue fetching and bulk operations
- Jira MCP connection or REST API credentials (for Jira)
