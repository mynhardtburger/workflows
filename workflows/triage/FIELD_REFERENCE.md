# Triage Workflow Field Reference

This document describes all available fields and configuration options for the triage workflow.

## Architecture

The triage workflow separates **data generation** from **report rendering**:

1. **AI agent** analyzes issues and produces `triaged_issues.json` (structured data)
2. **`render-report.py`** reads the JSON + templates and outputs `report.html` + `triage-report.md`
3. **`bulk-operations.sh`** executes approved operations from the HTML report

Templates are never modified or regenerated by the AI agent.

## Workflow Configuration (.ambient/ambient.json)

### Core Fields

| Field | Type | Required | Description |
|-------|------|----------|-------------|
| `name` | string | Yes | Display name of the workflow |
| `description` | string | Yes | Brief description of what the workflow does |
| `systemPrompt` | string | Yes | Instructions for the AI assistant on how to execute the workflow |
| `startupPrompt` | string | No | Welcome message shown when the workflow starts |
| `results` | object | No | Map of result categories to file path patterns |

### Results Configuration

```json
{
  "results": {
    "Triage Data": "artifacts/triage/triaged_issues.json",
    "Interactive Report": "artifacts/triage/report.html",
    "Triage Report": "artifacts/triage/triage-report.md"
  }
}
```

## Data Schema: triaged_issues.json

The AI agent's only data output. This file drives all report rendering.

### Top-Level Structure

```json
{
  "metadata": { ... },
  "issues": [ ... ]
}
```

### Metadata Fields

| Field | Type | Required | Description | Example |
|-------|------|----------|-------------|---------|
| `source` | string | Yes | Issue tracking platform | `"github"`, `"jira"` |
| `url` | string | Yes | Repository or project URL | `"https://github.com/owner/repo"` |
| `name` | string | Yes | Repository or project name | `"owner/repo"`, `"RHOAIENG"` |
| `jql` | string | No | JQL query used (Jira only) | `"project = X AND status = Open"` |
| `date` | string | Yes | Generation date (YYYY-MM-DD) | `"2026-03-23"` |
| `totalIssues` | integer | Yes | Total issue count | `42` |

### Issue Fields

| Field | Type | Required | Description | Example |
|-------|------|----------|-------------|---------|
| `number` | string | Yes | Issue number or Jira key | `"42"`, `"RHOAIENG-1234"` |
| `title` | string | Yes | Issue title | `"Fix authentication bug"` |
| `url` | string | Yes | Link to the issue | `"https://github.com/owner/repo/issues/42"` |
| `type` | string | Yes | Issue type | `"bug"`, `"feature"`, `"enhancement"`, `"documentation"` |
| `priority` | string | Yes | Priority level | `"critical"`, `"high"`, `"medium"`, `"low"` |
| `status` | string | Yes | Current issue status | `"open"`, `"Backlog"`, `"New"` |
| `recommendation` | string | Yes | Triage recommendation | See Recommendations section |
| `reason` | string | Yes | Explanation for recommendation | Free text |
| `waitingOn` | string | Yes | Who/what is blocking progress | `"username"`, `"-"` |
| `lastModifiedBy` | string | Yes | Username who last updated | `"octocat"` |
| `lastModifiedDate` | string | Yes | ISO 8601 timestamp | `"2026-01-19T12:34:56Z"` |
| `nextAction` | string | Yes | Recommended next step | `"Close as duplicate of #1234"` |
| `customLabels` | array | No | Extra labels to apply | `["ai-fixable", "ai-triaged"]` |

## Recommendations

### Available Recommendation Types

| Recommendation | Badge Color | Description | Default Operation |
|----------------|-------------|-------------|-------------------|
| `CLOSE` | Red | Invalid, obsolete, or resolved | Close issue with reason |
| `FIX_NOW` | Yellow | Critical or quick win | Add `priority:high` label |
| `BACKLOG` | Blue | Valid for future work | Add `backlog` label |
| `NEEDS_INFO` | Pink | Blocked on information | Add `needs-info` label |
| `DUPLICATE` | Gray | Duplicate of existing issue | Link to original and close |
| `CBA_AUTO` | Green | Can be automated by agent | Add `cba:auto-fix` label |
| `ASSIGN` | Purple | Ready for assignment | Add `ready` label |
| `WONT_FIX` | Gray | Valid but won't address | Close with reason |

### Recommendation Logic

The AI considers:

- **Age**: How long has the issue been open?
- **Activity**: Recent comments or updates?
- **Labels**: Existing priority/type labels
- **Assignees**: Currently assigned?
- **Duplicates**: Similar to other issues?
- **Feasibility**: Can a codebase agent automate it?
- **Impact**: How many users affected?
- **Complexity**: Simple fix vs major feature?

## Render Script

The `scripts/render-report.py` script performs deterministic placeholder replacement.

### Usage

```bash
python3 scripts/render-report.py \
  --data artifacts/triage/triaged_issues.json \
  --output artifacts/triage/ \
  --templates templates/
```

### Arguments

| Argument | Required | Description |
|----------|----------|-------------|
| `--data` | Yes | Path to `triaged_issues.json` |
| `--output` | Yes | Output directory for rendered reports |
| `--templates` | Yes | Directory containing `report.html` and `triage-report.md` templates |

### HTML Template Placeholders

| Placeholder | Description | Example |
|-------------|-------------|---------|
| `{REPO_URL}` | Repository/project URL | `https://github.com/owner/repo` |
| `{REPO_NAME}` | Repository/project name | `owner/repo` |
| `{DATE}` | Generation date | `2026-03-23` |
| `{TOTAL_ISSUES}` | Total issue count | `62` |
| `{CLOSE_COUNT}` | Issues recommended to close | `5` |
| `{FIX_NOW_COUNT}` | Issues to fix now | `28` |
| `{BACKLOG_COUNT}` | Issues for backlog | `25` |
| `{NEEDS_INFO_COUNT}` | Issues needing info | `4` |
| `{AMBER_AUTO_COUNT}` | Issues for CBA automation | `0` |
| `{ASSIGN_COUNT}` | Issues to assign | `0` |
| `{DUPLICATE_COUNT}` | Duplicate issues | `3` |
| `{WONT_FIX_COUNT}` | Issues marked won't fix | `2` |
| `{ISSUES_JSON}` | JSON array of all issues | `[{...}, {...}]` |
| `{METADATA_JSON}` | JSON metadata object | `{...}` |
| `{TABLE_ROWS}` | Empty (table rendered by JS) | `` |

## Bulk Operations

### Operations JSON Format

The HTML report generates this JSON when users select issues and click "Generate Operations Script":

```json
[
  {
    "issue": "1234",
    "action": "CLOSE",
    "params": {
      "reason": "Duplicate of #5678"
    }
  },
  {
    "issue": "RHOAIENG-5678",
    "action": "ADD_LABEL",
    "params": {
      "labels": ["priority:high", "ai-triaged"]
    }
  }
]
```

### Available Actions

| Action | Params | Description |
|--------|--------|-------------|
| `CLOSE` | `reason` | Close issue with comment |
| `ADD_LABEL` | `labels` (array) | Add one or more labels |
| `REMOVE_LABEL` | `labels` (array) | Remove one or more labels |
| `ASSIGN` | `assignee` | Assign to user |
| `COMMENT` | `comment` | Add comment |
| `LINK_DUPLICATE` | `original` | Link as duplicate and close |

### Bulk Operations Script

```bash
# GitHub
./scripts/bulk-operations.sh --backend github --repo owner/repo --operations operations.json

# Jira
./scripts/bulk-operations.sh --backend jira --operations operations.json

# Dry run (preview without making changes)
./scripts/bulk-operations.sh --dry-run --backend github --repo owner/repo --operations operations.json
```

### Environment Variables

| Variable | Backend | Description |
|----------|---------|-------------|
| `GITHUB_TOKEN` | GitHub | Personal access token |
| `JIRA_URL` | Jira | Jira instance URL |
| `JIRA_EMAIL` | Jira | Jira user email |
| `JIRA_API_TOKEN` | Jira | Jira API token |

## GitHub Actions Configuration

### Workflow Triggers

```yaml
on:
  schedule:
    - cron: '0 0 1 * *'  # Monthly on 1st at 00:00 UTC
  workflow_dispatch:      # Manual trigger
    inputs:
      repository:         # Optional: override repo
        description: 'Repository to triage (owner/repo format)'
        required: false
```

### Workflow Outputs

| Artifact | Description | Retention |
|----------|-------------|-----------|
| `triage-report-{run_number}` | ZIP with all reports | 90 days |

Contents:

- `report.html` - Interactive dashboard
- `triage-report.md` - Markdown report
- `triaged_issues.json` - Raw JSON data

## Customization

### Add New Recommendation Type

1. Add badge style to `templates/report.html`:

```css
.badge-my-type { background: #color; color: #text-color; }
```

2. Add stat card to HTML template
3. Add operation mapping in `generateOperations()` JS function
4. Update `render-report.py` to compute the new stat count
5. Update this field reference

### Custom Labels

Use the `customLabels` field in `triaged_issues.json` to add arbitrary labels per issue. These are included as additional `ADD_LABEL` operations in the generated script.

### Change Sort Order

Edit `templates/report.html`:

```javascript
let currentSort = { field: 'priority', direction: 'desc' };
```

## Troubleshooting

### Common Issues

**Render script fails**

- Ensure `triaged_issues.json` has both `metadata` and `issues` keys
- Check that template files exist at the specified `--templates` path
- Validate JSON syntax of the data file

**Operations fail**

- Verify GitHub token has correct permissions
- Check rate limiting
- Ensure issue numbers are valid
- Review operation JSON syntax
- For Jira: verify JIRA_URL, JIRA_EMAIL, JIRA_API_TOKEN are set

**Badge colors wrong**

- Ensure recommendation values match exactly (case-sensitive)
- Check that CSS class exists for the recommendation type
